# 学术级报告统一规范

本文档是五篇报告的统一工艺标准。所有专题报告必须遵守本规范，不得输出半成品、占位段落、空表格、伪公式或无法解释的图片堆叠。

## 一、总目标

五篇报告均应达到“中文学术报告成品”水平，兼顾以下目标：

1. 论证完整：必须覆盖研究背景、预训练模型介绍、任务定义、训练策略、微调过程、实验设计、结果分析、可解释性分析、局限性与改进建议。
2. 证据可核验：本地指标必须来自仓库内脚本、日志、图像或配置；外部背景信息必须来自论文、官方仓库、模型卡或官方文档。
3. 视觉规范：Word 字体、段落、图题、表题、页边距、页眉页脚、图表编号、公式编号必须统一。
4. 中文风格严谨：避免口语化、营销化、空泛形容。结论必须建立在可证据支撑的事实上。
5. 文档可直接交付：必须输出最终 `.docx`，并确保打开正常、中文不乱码、插图不变形、分页基本合理。

## 二、五篇报告与主题边界

1. `languagebind_imagebind`：LanguageBind 与 ImageBind 对比报告。必须把两者放在统一框架下比较，包括原始预训练思想、共享嵌入空间机制、特征提取流程、联合分类策略、实验对比和路线调整原因。
2. `dinov3`：DINOv3 蒸馏版图像模型报告。聚焦 DINOv3 ConvNeXt-Tiny 的预训练来源、蒸馏关系、下游微调结构、训练细节、图像端性能与可解释性。
3. `cedmini`：CED-Mini 音频模型报告。聚焦音频表征、Mel 频谱处理、SpecAugment、渐进解冻、TTA、音频端结果与 Mel-GradCAM 分析。
4. `dataset_preprocessing`：数据集与预处理报告。聚焦数据来源、类别体系、样本统计、YOLO 裁剪流程、音频特征准备、质量控制与潜在偏差。
5. `deployment`：部署与系统实现报告。聚焦 Gradio 原型、推理链路、统一状态空间、语义映射矩阵、风险提示机制、系统输入输出和工程约束。

## 三、统一写作结构

每篇报告至少包含以下章节，可根据主题微调章节标题，但逻辑顺序不可破坏：

1. 摘要
2. 关键词
3. 研究背景与问题定义
4. 原始预训练模型介绍
5. 本项目任务建模与方法设计
6. 数据集与预处理
7. 训练策略与微调过程
8. 实验设置与评价指标
9. 结果与分析
10. 可解释性分析
11. 局限性与改进方向
12. 结论
13. 参考文献

如果专题需要，可增加“部署实现”“系统架构”“对比讨论”等章节。

## 四、格式规范

### 1. 页面与字体

1. 页面：A4 纵向，常规页边距，正文单栏。
2. 中文正文字体：宋体。
3. 中文一级、二级标题：黑体。
4. 英文与数字：Times New Roman。
5. 正文字号：小四附近视觉等价。
6. 行距：固定值约 24pt 或视觉等价的学术文档行距。
7. 首行缩进：正文段落 2 字符左右。

### 2. 封面与前置部分

1. 每篇报告都要有独立封面。
2. 封面必须包含：报告标题、副标题或专题说明、项目名称、生成日期。
3. 摘要与关键词单独成节。

### 3. 标题层级

1. 一级标题示例：`1 引言`
2. 二级标题示例：`1.1 研究背景`
3. 三级标题可用 `1.1.1`
4. 不使用花哨颜色标题，不使用无编号散乱标题。

### 4. 图表规范

1. 所有图均需图题，格式为“图 1-1 ×××”或同等清晰编号。
2. 所有表均需表题，格式为“表 1-1 ×××”或同等清晰编号。
3. 图题置于图下方，表题置于表上方或下方，但全文统一。
4. 图后必须有文字分析，不能只贴图不解释。
5. 表格内容必须有列标题，单位或百分比要明确。
6. 插图优先使用本地已有训练曲线、混淆矩阵、ROC、PR/F1、Grad-CAM、t-SNE 等。
7. 流程图与架构图优先使用 draw.io 原生 `.drawio` 生成，再导出为 PNG/SVG 插入。

### 5. 公式规范

1. 每篇至少给出与主题强相关的关键公式，禁止为凑数堆砌无关公式。
2. 公式必须编号，并在正文解释变量含义。
3. 可采用高质量公式图片或清晰的 Word 公式块，但要避免乱码和基线错位。
4. 典型可用公式包括：
   - softmax 概率定义
   - focal loss
   - mixup/cutmix 标签混合
   - EMA 更新
   - 加权融合公式
   - macro-F1、AUC 或 balanced 指标定义

### 6. 参考文献规范

1. 参考文献必须区分“本地实验依据”和“外部学术/官方依据”。
2. 外部引用必须优先使用论文、官方仓库、官方模型卡、官方博客或官方文档。
3. 引文应在文末给出标题、作者或机构、年份、URL。
4. 不引用二手搬运内容，不引用无出处论坛总结作为核心依据。

## 五、内容深度要求

### 1. 预训练模型介绍

必须说明：

1. 模型提出背景与论文/仓库来源。
2. 预训练范式或核心目标。
3. 输入模态、表示空间和典型能力。
4. 与本项目采用方式的关系。
5. 为什么适合或不适合作为最终部署主干。

### 2. 训练与微调过程

必须具体到：

1. 输入特征形式。
2. 主干冻结/解冻策略。
3. 分类头或投影头结构。
4. 学习率与调度器。
5. 损失函数。
6. 数据采样策略。
7. 增强策略。
8. EMA / TTA / Mixup / SpecAugment / Grad-CAM 等是否使用、为何使用。
9. 早停指标如何定义。

### 3. 结果分析

必须包含：

1. 核心指标表。
2. 曲线或混淆矩阵等可视化。
3. 各任务或各类别表现差异解释。
4. 对优势与短板的归因分析。
5. 与前一阶段或对照模型的比较。

### 4. 可解释性分析

至少覆盖其中一类，并优先使用本地现有图：

1. Grad-CAM 或 Mel-GradCAM
2. t-SNE 特征分布
3. ROC、PR/F1、置信度分析
4. Top confusion 误分类分析

### 5. 局限与讨论

必须写明真实问题，例如：

1. 类别不平衡
2. 猫图像样本量较小
3. 图像与音频标签体系异构
4. 多模态样本并非严格配对
5. 原型已完成但真实在线部署还需验证

## 七、图文组织建议

### 1. 对比报告

LanguageBind 与 ImageBind 报告建议至少包含：

1. 一个共享嵌入空间/训练流程示意图
2. 一张总指标对比表
3. 两组训练曲线或解释性图
4. 一节“为何后续转向 DINOv3 + CED-Mini 部署路线”的讨论

### 2. 单模型报告

DINOv3 与 CED-Mini 报告建议至少包含：

1. 一个模型结构或训练流程图
2. 一个核心超参数表
3. 一个主结果表
4. 两到三张本地解释性或结果图

### 3. 数据与部署报告

数据报告必须包含样本统计表与预处理流程图。

部署报告必须包含：

1. 系统架构图
2. 推理流程图
3. 统一状态空间与语义映射解释
4. 工程输入输出与风险提示机制

## 八、质量底线

以下情况视为不合格：

1. 文档中出现“待补充”“TODO”“此处略”等占位内容。
2. 图题、表题、编号缺失。
3. 公式乱码、错位或无法辨认。
4. 引用指标与日志不一致。
5. 结论无法回到证据。
6. 中文学术风格明显不足，像博客或随手笔记。

## 九、交付要求

每个专题目录至少包含：

1. 一份最终 `.docx`
2. 一个生成脚本
3. 一个素材/引文说明文件，记录用到的本地文件与外部来源
4. 若新建流程图，保留 `.drawio` 源文件及导出图

五篇最终文档统一输出到 `output/doc/`。
